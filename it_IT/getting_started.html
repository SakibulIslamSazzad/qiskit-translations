Come Iniziare con Qiskit¶

Il flusso di lavoro per l’utilizzo di Qiskit consiste in tre fasi di alto livello:

  • Costruire: disegna un circuito quantistico che rappresenta il problema preso in considerazione.

  • Eseguire: esegui esperimenti su diversi backend (che includono sia sistemi reali che simulatori).

  • Analizzare: calcola statistiche riassuntive e visualizzare i risultati degli esperimenti.

Ecco un esempio dell’intero flusso di lavoro, dove ogni passaggio sarà dettagliato nelle sezione successive:

import numpy as np
from qiskit import(
  QuantumCircuit,
  execute,
  Aer)
from qiskit.visualization import plot_histogram

# Use Aer's qasm_simulator
simulator = Aer.get_backend('qasm_simulator')

# Create a Quantum Circuit acting on the q register
circuit = QuantumCircuit(2, 2)

# Add a H gate on qubit 0
circuit.h(0)

# Add a CX (CNOT) gate on control qubit 0 and target qubit 1
circuit.cx(0, 1)

# Map the quantum measurement to the classical bits
circuit.measure([0,1], [0,1])

# Execute the circuit on the qasm simulator
job = execute(circuit, simulator, shots=1000)

# Grab results from the job
result = job.result()

# Returns counts
counts = result.get_counts(circuit)
print("\nTotal count for 00 and 11 are:",counts)

# Draw the circuit
circuit.draw()

Flusso di Lavoro Passo dopo Passo¶

Il programma sopra riportato può essere suddiviso in sei passaggi:

  1. Importare i pacchetti

  2. Inizializzare le variabili

  3. Aggiungere i gate

  4. Visualizzare il circuito

  5. Simulare l’esperimento

  6. Visualizzare i risultati

Passo 1: Importare i Pacchetti¶

Gli elementi di base necessari per il tuo programma si importano nel modo seguente:

import numpy as np
from qiskit import(
  QuantumCircuit,
  execute,
  Aer)
from qiskit.visualization import plot_histogram

Più in dettaglio, le istruzioni di import sono

  • QuantumCircuit: può essere considerato l’insieme delle istruzioni del sistema quantistico. Contiene tutte le operazioni quantistiche.

  • execute: esegue il tuo circuito / esperimento.

  • Aer: si occupa dei backend per le simulazioni.

  • plot_histogram: crea istogrammi.

Passo 2 : Inizializzare le variabili¶

Considera la seguente linea di codice

circuit = QuantumCircuit(2, 2)

Qui si stanno inizializzando 2 qubit nello stato zero; con 2 bit classici impostati a zero; e circuit è il circuito quantistico.

Sintassi:

  • QuantumCircuit(int, int)

Passo 3: Aggiungere i Gate¶

Puoi aggiungere gate (operazioni) per manipolare i registri del tuo circuito.

Considera le seguenti tre linee di codice:

circuit.h(0)
circuit.cx(0, 1)
circuit.measure([0,1], [0,1])

I gate sono aggiunti al circuito uno ad uno per formare lo stato di Bell

\[|\psi\rangle = \left(|00\rangle+|11\rangle\right)/\sqrt{2}.\]

Il codice sopra riportato applica i seguenti gate:

  • QuantumCircuit.h(0): Un gate Hadamard \(H\) sul qubit 0, che lo pone in uno stato di sovrapposizione.

  • QuantumCircuit.cx(0, 1): Un’operazione controlled-Not (\(C_{X}\)) sul qubit di controllo 0 e qubit target 1, che pone i qubit in uno stato di entanglement.

  • QuantumCircuit.measure([0,1], [0,1]): quando si passa il registro quantistico ed il registro classico a measure, il risultato della misurazione del qubit nella posizione i verrà salvato nel bit classico in posizione i.

Passo 4 : Visualizzare il Circuito¶

Puoi usare QuantumCircuit.draw() per visualizzare il circuito che hai disegnato nelle varie forme utilizzate in molti testi ed articoli di ricerca.

circuit.draw()

In questo circuito, i qubit sono ordinati con il qubit zero nella parte alta del circuito ed il qubit uno in basso. Il circuito si legge da sinistra verso destra, ovvero i gate che vengono applicati prima nel circuito appaiono più a sinistra.

Il backend predefinito per QuantumCircuit.draw() o qiskit.visualization.circuit_drawer() è il backend text. Tuttavia, a seconda del tuo ambiente locale potresti voler modificare il default in qualcosa di più adatto ad un caso d’uso specifico. È possibile farlo modificando il file di configurazione utente. Il percorso predefinito del file di configurazione è ~/qiskit/settings.conf ed è un file di tipo .ini.

Un esempio di file settings.conf per impostare un drawer Matplotlib è:

[default]
circuit_drawer = mpl

Puoi usare qualsiasi backend drawer valido come valore per questa impostazione, come text, mpl, latex, e latex_source.

Passo 5: Simulare l’Esperimento¶

Qiskit Aer è un framework di simulazione per circuiti quantistici ad alta prestazione. Mette a disposizione diversi backend che permettono di effettuare simulazioni per diversi scopi.

Se si incontrano problemi nell’installazione di Aer, in alternativa si può utilizzare il provider Basic Aer semplicemente sostituendo Aer con BasicAer. Basic Aer è incluso in Qiskit Terra.

import numpy as np
from qiskit import(
  QuantumCircuit,
  execute,
  BasicAer)
...

Per simulare questo circuito, userai il qasm_simulator. Ogni esecuzione di questo circuito produrrà come risultato la stringa di bit 00 o 11.

simulator = Aer.get_backend('qasm_simulator')
job = execute(circuit, simulator, shots=1000)
result = job.result()
counts = result.get_counts(circuit)
print("\nTotal count for 00 and 11 are:",counts)


Total count for 00 and 11 are: {'00': 516, '11': 484}

Come previsto, la stringa di bit risultante è 00 per circa il 50 percento delle volte. Il numero di volte che il circuito viene eseguito può essere specificato utilizzando l’argomento shots del metodo execute. Il numero di shot della simulazione era impostato su 1000 (il valore predefinito è 1024).

Una volta che hai un oggetto result, puoi accedere ai conteggi utilizzando il metodo get_counts(circuit). Questo fornirà i risultati aggregati dell’esperimento eseguito.

Passo 6 : Visualizzare i Risultati¶

Qiskit fornisce diverse tipologie di visualizzazione,

inclusa la funzione plot_histogram per visualizzare i tuoi risultati.

plot_histogram(counts)

Le probabilità osservate \(Pr(00)\) and \(Pr(11)\) sono calcolate dividendo il conteggio rispettivo per il numero totale di shot.

Nota

Prova a modificare il valore la parola chiave shots nel metodo execute per vedere come cambiano le probabilità risultanti.
